# Project Structure

## Directory Layout

```
html2pdf/
├── .github/
│   └── workflows/
│       ├── ci.yml              # PR checks (lint + integration tests)
│       └── publish.yml         # Release + Docker Hub publish
├── src/
│   ├── config.js               # Environment variable parsing
│   ├── renderer.js             # Chrome lifecycle + PDF generation
│   ├── schemas.js              # JSON Schema for /render endpoint
│   └── server.js               # Fastify app, routes, error handler
├── .dockerignore               # Files excluded from Docker build
├── .gitignore                  # node_modules, npm-debug.log, .env
├── CHANGELOG.md                # Auto-generated by release-please
├── docker-compose.yml          # Local development / deployment
├── Dockerfile                  # Multi-stage prod image
├── eslint.config.js            # ESLint flat config (ES2024)
├── LICENSE                     # MIT
├── package.json                # Project metadata + deps
├── package-lock.json           # Deterministic installs
└── README.md                   # API docs + usage examples
```

## Key Locations

| What                      | Where                              |
| ------------------------- | ---------------------------------- |
| Application entry point   | `src/server.js`                    |
| PDF rendering core        | `src/renderer.js`                  |
| Request validation        | `src/schemas.js`                   |
| Environment config        | `src/config.js`                    |
| CI pipeline               | `.github/workflows/ci.yml`         |
| Release pipeline          | `.github/workflows/publish.yml`    |
| Docker image def          | `Dockerfile`                       |
| Docker local dev          | `docker-compose.yml`               |

## Naming Conventions

| Convention        | Example                        | Notes                         |
| ----------------- | ------------------------------ | ----------------------------- |
| Source files      | `renderer.js`, `schemas.js`    | Lowercase, singular nouns     |
| Config file       | `config.js`                    | Single flat config object     |
| Entry point       | `server.js`                    | Named after Fastify role      |
| Workflows         | `ci.yml`, `publish.yml`        | Short, descriptive names      |
| Docker files      | `Dockerfile`, `docker-compose.yml` | Standard Docker conventions |

## Module Dependencies

```
server.js
├── config.js       (configuration)
├── renderer.js     (exports: render, healthCheck, shutdown)
│   ├── config.js   (Chrome args, concurrency, timeout)
│   ├── puppeteer-core
│   └── pdf-lib
└── schemas.js      (exports: renderSchema)
```

All imports use ESM (`import`). Internal imports use relative paths with `.js` extensions.

## What's NOT in the codebase

- No `test/` directory (tests are inline in CI workflow as shell commands)
- No `types/` or TypeScript files
- No middleware files
- No database migrations
- No `.env.example` (config is documented in README)
- No monorepo structure — single flat package
